Chapter 09

Generating Text

How the model actually writes — one token at a time, guided by probabilities and controlled by temperature.

Now for the fun part. We've seen how the model is trained — predict next tokens, calculate loss, update weights. But how does it actually generate text once trained?

The Generation Loop

Generation is surprisingly simple. It uses the exact same machinery as training, but instead of calculating loss and updating weights, we pick a token from the probability distribution and add it to the sequence:

① Start: Feed prompt tokens into model
↓
② Model outputs probability for every possible next token
↓
③ Pick one token from the probability distribution
↓
④ Add picked token to the sequence
↓ repeat from ①

The top 5 add up to over 99%. That leaves only 1% spread across all 65,531 other tokens. The model is very confident that "Paris" comes next.

We pick "Paris" (98.1% chance), add it to the sequence, and now our input is "The capital of France is Paris". We feed this back in and the model predicts what comes after "Paris". Maybe "." (period) or "," or ", which" — and so the generation continues, one token at a time, until the model outputs a stop token or reaches a length limit.

Greedy vs. Sampling

If we always pick the most likely token (called greedy decoding), then the same prompt will always generate the exact same text. But you know from using ChatGPT that it often gives different answers to the same question. That's because instead of always picking the top choice, the model samples randomly according to the probabilities.

Greedy Decoding

Always pick the highest probability token. Deterministic — same input, same output. Good for factual queries.

Random Sampling

Pick randomly weighted by probability. Creative and varied. Good for writing, brainstorming, storytelling.

With "Paris is in" as input, the model might give: "France" 40%, "the" 15%, "a" 12%, "Europe" 8%, "northern" 6%... With sampling, sometimes you get "France" and sometimes "Europe" or "the heart of..." — leading to completely different continuations.

Temperature: Controlling Creativity

Temperature is a number that controls how random the sampling is. Think of molecules — at low temperatures, they're frozen and predictable; at high temperatures, they bounce around chaotically.

How Temperature Works

Low temperature (0.1–0.5): The probability distribution gets "sharpened" — the most likely token becomes overwhelmingly dominant. Almost like greedy decoding. Use for factual answers.

Temperature = 1.0: The original probabilities are used as-is. The default.

High temperature (1.5–3.0): The distribution gets "flattened" — less likely tokens get more chance. More creative and surprising, but also more likely to produce nonsense.

Mathematically, temperature divides the logits (the raw numbers before softmax) by T. When T is small, the differences between logits get amplified; when T is large, they get compressed.

Generation Keeps Going

Let's follow the generation further. Say we picked "also" after "The capital of France is":

🌳
Generation Tree
"The capital of France is Paris" (98.1%)
→ "Paris." (62%) → end
→ "Paris," (31%) → " which..." / " a city..."
"The capital of France is also" (0.5%)
→ "also known" (28%) → " as Paris" / " for its..."
→ "also the" (22%) → " country's..." / " largest..."
→ "also Paris" (18%)
"The capital of France is the" (0.7%)
→ "the city" (35%) → " of Paris..."
→ "the Paris" (19%)

The model really wants to get "Paris" in there — you can feel this "urge" it has to output "Paris" even in positions where it doesn't quite fit grammatically.

Tamil Example

Tamil Generation

Prompt: "தமிழ்நாட்டின் தலைநகரம்" (The capital of Tamil Nadu)

Top predictions:

சென்னை (Chennai) — ~95%

என்பது (is) — ~2%

ஆகும் (is/becomes) — ~1.5%

The model learned from Tamil text that Chennai overwhelmingly follows "capital of Tamil Nadu" — just like "Paris" follows "capital of France" in English.

One Wasteful Detail

During generation, the model processes the entire sequence each time we add a new token. When we input "The capital of France is Paris,", the model computes predictions for every position, but we only use the prediction for the last position. All the earlier computations are wasted — we already knew the predictions for "The", "capital", etc.

This is wildly inefficient. In Chapter 17, we'll see the KV Cache optimization that stores the intermediate calculations from previous tokens so they don't need to be recomputed. This makes generation much faster.

Key Insight

Text generation is just next-token prediction in a loop. The model predicts a probability distribution over all possible next tokens, we pick one (using greedy decoding, sampling, or temperature-adjusted sampling), add it to the sequence, and repeat. The same model architecture handles both training and generation — the only difference is whether we calculate loss and update weights (training) or pick tokens and grow the sequence (generation).
