Chapter 25

Mid-Training

Phase 2: Shifting to higher-quality data — more math, more code, more reasoning, more Tamil.

After base training, the model understands language generally. Mid-training refines it with a different, higher-quality data mix. Think of it like a student who first reads everything they can find (base training) and then focuses on textbooks and practice problems (mid-training).

What Changes

📐

More Math

Math textbooks, problems with worked solutions, competition math

💻

More Code

Higher quality code repositories, documentation, coding tutorials

🌍

More Languages

Increase representation of Tamil, Hindi, and other languages beyond English

📖

Higher Quality

Filter out low-quality web text, keep curated sources, encyclopaedias, textbooks

The learning rate is annealed (gradually reduced) during mid-training — we're fine-tuning what the model already knows rather than learning from scratch. Benchmark scores on math, code, and reasoning improve noticeably during this phase.

Tamil in Training

The training corpus includes Tamil text from news sites, Wikipedia Tamil edition, Tamil literature, and web forums. This is how the model learns to generate text like தமிழ்நாட்டின் தலைநகரம் சென்னை (The capital of Tamil Nadu is Chennai) — it has seen enough Tamil text to learn the patterns of the language.

Key Insight

Mid-training shifts the data mix toward quality over quantity. The model already knows language basics from base training; now it needs deeper knowledge in math, code, reasoning, and multilingual understanding. The learning rate is lowered so these refinements don't disrupt what was already learned.
