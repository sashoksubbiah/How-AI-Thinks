Chapter 13

Transformer Block

Inside the repeating unit that makes up the bulk of the model — attention for mixing between positions, MLP for thinking within each position.

The embedding module turned our 5 tokens into a 5×1280 matrix. Now this flows through 20 identical transformer blocks. Each block takes in T×D and outputs T×D. Let's look inside.

Anatomy of a Transformer Block

Input: T × D
↓
RMS Norm
↓
Causal Self-Attention (tokens talk to each other)
↓ + skip connection (add input back)
RMS Norm
↓
MLP (each token thinks independently)
↓ + skip connection (add back again)
Output: T × D

Two main components: Causal Self-Attention (the blue box — where tokens pull in information from earlier positions) and MLP (the purple box — where each token thinks independently). Each has a skip connection (Chapter 12) and RMS Normalisation (Chapter 22).

The MLP: Thinking Within Each Position

The MLP is our old friend from Chapter 10 — the linear-ReLU-linear sandwich. But inside the transformer, the dimensions are specific:

D = 1280
→
Linear: D → 4D (= 5120)
→
ReLU²
→
Linear: 4D → D (= 1280)
→
D = 1280

The key idea: expand from D to 4D, apply nonlinearity, then contract back to D. This expansion gives the model room to compute complex things before squeezing back down. Think of it like writing a draft four times longer than your final essay, then condensing it.

The MLP operates identically and independently on each position (row). If position 1 and position 3 have the same input vector, they'll get the same output. The MLP never mixes information between tokens — that's attention's job.

Worked Example (D=2)

🔢
MLP with D=2, 4D=8, T=3
Input (3×2): [ [1.00, 2.00], [-0.50, 1.00], [1.00, 2.00] ]
↓ Linear (2→8): multiply by 2×8 weight matrix
After Linear: (3×8) — each row is now 8 numbers
↓ ReLU: negative numbers → 0
After ReLU: (3×8) — some numbers became 0
↓ Linear (8→2): compress back to 2 numbers per position
Output (3×2): [ [0.35, -0.78], [0.12, 0.44], [0.35, -0.78] ]

Notice: positions 1 and 3 have the same input [1.00, 2.00], so they get the same output [0.35, -0.78]. The MLP treats each position independently.

The Attention Layer: Mixing Between Positions

The MLP handles "thinking within each position." But to predict the next word after "The capital of France is", the model needs the "is" position to pull in information from "capital" and "France" — positions that are far away. This is what causal self-attention does: it lets each position look at all earlier positions and decide which information to bring in.

Attention is so important and complex that it gets its own chapter — next up.

What About RMS Norm?

RMS Normalisation rescales the numbers at each layer to keep them in a healthy range (preventing numbers from exploding or vanishing as they pass through many layers). It's like an automatic volume control. We'll cover the exact calculation in Chapter 22. For now, just know it has no learned parameters in our model — it's a fixed mathematical operation.

Key Insight

Each transformer block has two halves: Attention (tokens look at each other — mixing information across positions) and MLP (each token processes its information independently — the linear→ReLU→linear sandwich). Both halves use skip connections. The MLP alone has 13.1 million parameters per block — across 20 blocks, that's 262 million parameters, and it's where a huge amount of learning happens. But it's the attention mechanism that makes the transformer truly special.
