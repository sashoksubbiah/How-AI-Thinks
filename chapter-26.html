Chapter 26

Tool Use: The Tool Dance

How AI models reach beyond text to search the web, run code, and call APIs — using nothing but token prediction.

A language model generates tokens. But modern AI assistants can search the web, run Python code, check the weather, and more. How? Through an elegant trick: special tokens that signal tool use.

The Tool Dance

User: "சென்னையின் வானிலை எப்படி?" (What's the weather in Chennai?)
↓ model generates tokens...
Model outputs: <tool_call> weather(location="Chennai") </tool_call>
↓ system intercepts, runs the tool
System: <tool_result> {"temp": "32°C", "condition": "Sunny"} </tool_result>
↓ result added to context, model continues
Model: "சென்னையில் இப்போது 32°C, வெயில் அடிக்கிறது!" (It's 32°C and sunny in Chennai!)

The key insight: the model doesn't "use" tools — it generates tokens that happen to be tool calls. The surrounding system parses these tokens, executes the tool, and feeds the result back as more tokens. To the model, it's all just next-token prediction.

Training the model to use tools requires showing it many examples of the tool dance format during training. It learns when tool use is appropriate, which tool to call, and how to incorporate tool results into its response.

Key Insight

Tool use is just token prediction with special tokens. The model generates <tool_call> tokens, the system executes the tool and returns results as <tool_result> tokens, and the model continues generating. No special architecture needed — it's all learned through training examples.
