Chapter 11

Cracking Open the Transformer

From token IDs to rich embeddings — the first step inside the model that turns meaningless numbers into meaning.

The Pandora Analogy

Remember the music streaming service Pandora? Every song had a database ID — say, 153,432 for "Love Yourself" by Justin Bieber. But an ID number tells you nothing useful. You can't recommend the next song based on "153,432."

Pandora's insight: hire musicians to characterise each song along hundreds of dimensions — genre, tempo, instrumentation, mood, vocal style. Now song 153,432 has rich, meaningful information attached to it. Songs with similar descriptions can be recommended together.

We need to do the same thing with tokens. Token 6237 (" Paris") is just a number. But "Paris" carries rich meaning: city, France, Europe, beautiful, Eiffel Tower, romance. The first step inside our transformer is to expand each token ID into a rich vector of numbers — an embedding — that captures this meaning.

The Top-Level Architecture

Input: T token IDs (e.g., 5 tokens)
↓
Embedding Module → T × D matrix (e.g., 5 × 1280)
↓
🔁 Transformer Block × 20 layers (each: T×D → T×D)
↓
Final Linear Layer → T × V (e.g., 5 × 65,536)

The Dimension D

D is the embedding dimension — how many numbers represent each token. It's the single most important size in the model. Everything between the embedding module and the final output layer operates at dimension D.

D can't be too small — how could 4 numbers encode enough meaning to predict among 65,536 tokens? It also can't be too large relative to the model, or training won't converge.

The Embedding Module

Mechanically, the embedding is a lookup table. For each of the V = 65,536 tokens, we store D = 1,280 numbers. When token 6237 (" Paris") comes in, we look up row 6237 and get back 1,280 numbers.

Contrived Example (D=4)

"Paris" → [0.9, -0.2, -0.1, 0.8] ← noun, not adjective, not red, positive
"red"   → [0.1, 0.9, 0.9, 0.3] ← slightly noun, adjective, very red, slightly positive
"dog"   → [0.9, -0.3, -0.1, 0.7] ← noun, not adjective, not red, positive
"ugly"  → [-0.1, 0.8, 0.0, -0.7] ← not noun, adjective, not red, negative

In reality, D = 1,280 and the dimensions don't map to human concepts this neatly. The model figures out what each dimension should mean during training.

The crucial thing: the embedding values are learned during training. Initially they're random. Through backpropagation, tokens that appear in similar contexts develop similar embeddings. "Paris" and "London" end up close together in this 1,280-dimensional space; "Paris" and "because" end up far apart.

Vectors: The Language of Tensors

A vector is just a 1-dimensional list of numbers — like a single row of a matrix. When we say "the embedding of 'Paris' is a vector of size 1,280," we mean it's 1,280 numbers in a list. Think of each row of our T×D matrix as a vector representing one token.

After embedding, our 5-token input " He went to the" becomes a 5×1280 matrix — 5 vectors, each with 1,280 numbers full of meaning. This matrix flows through 20 transformer blocks, each outputting another 5×1280 matrix with increasingly refined understanding. Finally, the output layer converts each 1280-number vector into 65,536 probabilities.

Key Insight

The embedding module transforms meaningless token IDs into rich, 1280-dimensional vectors of meaning — learned automatically during training. Tokens that appear in similar contexts develop similar embeddings. This is the foundation upon which everything else in the transformer builds. With 84 million parameters just in embeddings, you can see why these models need so much data and compute to train.
