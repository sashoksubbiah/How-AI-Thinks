Chapter 05

The Matrix

Organizing numbers into matrices — the key to staying sane as models get bigger, and the reason GPUs rule the world.

Our turkey model had 2 weights and 3 turkeys, and already the diagrams in Chapter 4 were getting messy. What if we had 10 weights, 20 turkeys, more features like wingspan — and more complex calculations? We'd get confused immediately. We need a better way to stay organized.

🗂️

Key insight: A matrix is just a spreadsheet. Rows and columns of numbers, nothing magical. But by defining rules for how to multiply matrices together, we can express incredibly complex calculations cleanly. Think of it as upgrading from scattered sticky notes to a well-organized filing cabinet.

Organizing Our Turkey Data

Let's stick our turkey data into a matrix called X (by convention, X or x means the input data):

X (data: 3×2)
[ 1.00   1.50 ]
[ 0.75   1.25 ]
[ 1.25   1.00 ]
heights   lengths

Each row is a turkey. Each column is a feature (height, length). It's exactly the table from Chapter 3 without the headers.

The weights go into a matrix called W:

W (weights: 2×1)
[ 1000 ]
[ 3000 ]
w₁    w₂

Matrix Multiplication = Our Prediction

When we multiply X by W (written as X @ W or XW), we get exactly the predictions we calculated in Chapter 3:

Rule: An m×n matrix times an n×p matrix gives an m×p matrix. Here: 3×2 times 2×1 gives 3×1.

The Full Loss in Matrix Form

Instead of the sprawling equation from Chapter 3, we can now write:

loss = sum( (X @ W − Y_actual)² )

That's it. X is our turkey data, W is our weights, Y_actual is the actual feather counts, and the whole thing computes the loss. The forward calculation diagram goes from messy (Chapter 4) to clean:

The backward pass is similarly clean. Working backwards from loss, the gradient of W comes out as a matrix — what we now call the gradient (the matrix of all partial derivatives).

The Gradient

In Chapter 3 we computed the partial derivative of loss w.r.t. w₁ (got 1,875) and w.r.t. w₂ (got 3,500) separately. Now we can put them together as a gradient matrix:

Gradient of W
[ 1875 ]
[ 3500 ]

Update: W_new = W − learning_rate × gradient

The Power of Code

Here's the beautiful thing. In Python with the PyTorch library, all the head-spinning stuff — backpropagation, computing gradients — is done for you in seven lines of code:

import torch X = torch.tensor([[1.0, 1.5], [0.75, 1.25], [1.25, 1.0]]) Y_actual = torch.tensor([[5000.0], [3500.0], [4500.0]]) W = torch.tensor([[1000.0], [3000.0]], requires_grad=True) loss = ((X @ W - Y_actual) ** 2).sum() loss.backward() print(W.grad) # → tensor([[1875.], [3500.]])

We define the matrices, tell PyTorch we want the gradient of W, write the forward calculation, call loss.backward(), and the gradients appear. PyTorch tracks every calculation involving W, builds the computation graph, and does the backward pass automatically.

The full training loop for 1000 steps is equally compact. After running it, we get w₁ ≈ 2311 and w₂ ≈ 1633 — exactly matching our manual calculations from Chapter 3.

Why GPUs? Embarrassingly Parallel

Matrix multiplication has a remarkable property: each element of the result can be computed independently. Consider multiplying two 3×3 matrices:

ApproachOperationsSteps
Serial (one at a time)4545 steps
Parallel (9 processors)453 steps

Same total work, but 15× faster with parallelism. Scale this up to billions of operations and the difference is staggering.

This is why NVIDIA GPUs — originally designed for video game graphics (coloring millions of pixels independently) — turned out to be perfect for AI. An NVIDIA H100 GPU can perform nearly 2,000 trillion 16-bit floating point operations per second. That's not a typo — two quadrillion operations per second.

Brief History

In the early 2000s, NVIDIA realized their chips could do more than graphics. In 2007, they released CUDA, letting scientists use GPUs for general computation. AI researchers began experimenting with training neural networks on GPUs in the late 2000s. The real wake-up moment came in 2012 with AlexNet (more on this in Chapter 12), which used GPUs to crush an image recognition competition by an enormous margin. From that point forward, GPUs became essential to AI.

Key Insight

Matrices let us organize complex calculations cleanly. loss = sum((X @ W − Y_actual)²) expresses the entire turkey model in one line. The computer (specifically the GPU) can then execute all the operations in parallel. This combination of clean mathematical notation and massive parallelism is what makes training billion-parameter models possible.
