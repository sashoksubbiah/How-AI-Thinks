Chapter 21

Muon

A newer optimizer that thinks about weight matrices as a whole — not just individual parameters — to find better update directions.

Adam treats each parameter independently. But the weights in a linear layer form a matrix, and there may be better update directions when you consider the matrix as a whole.

The Key Idea: Matrix-Level Optimization

Muon uses a technique from linear algebra called the Singular Value Decomposition (SVD) to find the most effective direction to update an entire weight matrix at once. Think of it like this: Adam adjusts each knob on a mixing board independently, while Muon considers how all the knobs interact and finds coordinated adjustments that are more effective.

Specifically, Muon computes the gradient matrix, then uses an iterative approximation (Newton-Schulz iteration) to find a direction that "decorrelates" the gradient — removing redundant information and emphasising the most impactful update directions. This is computationally more expensive per step than Adam, but each step is more effective, so you need fewer total steps.

Adam

Updates each parameter independently. Simple, proven, widely used. The standard choice.

Muon

Updates weight matrices holistically. Better directions per step. Newer, used for cutting-edge models.

In practice, the model we'll train uses both: Adam for the embedding and final layer weights, Muon for the transformer block weights. This hybrid approach gets the best of both worlds.

Key Insight

Muon optimizes at the matrix level, finding better update directions by considering how all weights in a layer interact. Combined with Adam for other components, it represents the state of the art in transformer training optimization.
