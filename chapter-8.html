Chapter 08

What Exactly Does the Model Predict?

Understanding the inputs, outputs, and loss function of a GPT model — the three things you need to know before looking inside.

Our turkey model predicted a single number (feathers). But a GPT model predicts something far more complex: for every token in the input, it outputs a probability distribution over the entire vocabulary — a probability for every possible next token.

There are three questions we need to answer before we crack open the model:

What are X and Y?

What data goes in and what target are we comparing against?

How is loss calculated?

How do we get a single number that says "how wrong is the model?"

How does generation work?

How do you go from predictions to actual text? (Chapter 9)

Training on a Sentence

Let's go back to our tiny language and train on the text "he went to the store." We start by inputting just the beginning-of-sequence token <bos>. The model outputs a probability for every word in the vocabulary:

The model hasn't been trained yet, so these probabilities are essentially random. It thinks "bed" is most likely, but we want "he" — which currently has only 0.05% probability.

We don't care that the model is wrong right now — that's what training will fix. We continue feeding in tokens one by one, each time getting a prediction for the next token. After processing the full sequence, we have 5 predictions:

We only care about the probability the model assigned to the correct next token. Everything else is noise for loss calculation purposes.

Computing Loss

We need a single number that says "how wrong is the model?" — just like the loss in our turkey model. For language models, we multiply the probabilities of the correct tokens together:

Loss Calculation
1
Multiply probabilities: 0.05% × 2.48% × 5.72% × 4.72% × 40.54%
2
= 0.0005 × 0.0248 × 0.0572 × 0.0472 × 0.4054 = 0.0000000136
3
Negate (so lower = better): loss = −0.0000000136

In practice, we use the negative log of each probability instead of multiplying directly (this avoids numerical issues with multiplying tiny numbers and has nice mathematical properties). This is called cross-entropy loss:

Cross-Entropy Loss (What's Actually Used)
1
−log(0.0005) = 7.60
2
−log(0.0248) = 3.70
3
−log(0.0572) = 2.86
4
−log(0.0472) = 3.05
5
−log(0.4054) = 0.90
Σ
Average loss = (7.60 + 3.70 + 2.86 + 3.05 + 0.90) / 5 = 3.62

When probability is 100%, −log(1.0) = 0 (perfect, no loss). When probability is tiny, −log gives a large number (terrible, high loss). So lower loss = better predictions, just like our turkey model.

Feeding the Full Sequence at Once

I showed the model processing one token at a time for clarity, but in practice the entire sequence goes in at once. The model outputs probability distributions for all next tokens simultaneously. This is much more efficient because GPUs (Chapter 5) need big chunks of data to use their massive parallelism.

During actual training, over 60,000 tokens are fed to each GPU at once, and the GPU can make predictions on all 60,000, calculate loss, and adjust weights in less than half a second.

But Won't the Model Cheat?

If the whole sequence goes in at once, won't the model just peek at the next token instead of actually learning to predict it? Great instinct — models love to cheat. If they can lower the loss in any way, even unintended ways, they'll do it.

The transformer architecture prevents this through masking. When predicting the token after "went" in "He went to the store," the model literally cannot see "to" or any later tokens. The attention mechanism (Chapter 14) uses a mask that sets future token information to negative infinity, making it invisible. This is called causal masking — the model can only look backwards.

Batches and Tensors

In real training, we don't just feed one sequence — we feed a batch of many sequences simultaneously. If our batch size is B = 3 and sequence length is T = 5:

Input: B × T (3×5)
[ 0, 3, 11, 10, 9 ]
[ 0, 9, 4, 5, 7 ]
[ 0, 3, 5, 1, 6 ]
3 sequences × 5 tokens each
→ 🧠 →

The output is a 3-dimensional tensor of size B × T × V. "Tensor" just means "multi-dimensional array of numbers" — nothing scarier than a matrix, just with more dimensions.

The Training Concept in Full

The Whole Picture

We have text data (our corpus). We tokenize it into sequences of numbers. We feed batches of sequences into the model. The model outputs probability distributions for every next token. We compute loss by looking at how much probability the model gave to the correct next tokens. We backpropagate to compute gradients. We update all the weights. We repeat — millions of times.

If the model architecture is good enough (and the transformer is!), the loss will come down. The model will learn language, facts, reasoning, coding, and everything else present in its training data.

Key Insight

The loss function for language models looks only at the probability assigned to the correct next token — everything else is ignored for loss purposes. This feels strange at first, but it works: by maximizing the probability of correct next tokens across trillions of examples, the model learns the full richness of language. If 40% of the time "the" follows "went to" and 30% of the time "a" does, the model learns that distribution naturally.
