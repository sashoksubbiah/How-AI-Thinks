Chapter 19

Optimizers: Smarter Ways to Learn

Gradient descent was a great start, but training billion-parameter models needs much smarter weight update strategies.

The Problem with Basic Gradient Descent

In Chapter 3 we used the simplest possible approach: compute the gradient, multiply by a fixed learning rate, subtract from weights. This is called Stochastic Gradient Descent (SGD). It works for our 2-weight turkey model, but has serious problems at scale:

SGD Limitations

Fixed learning rate: Too small → takes forever. Too large → overshoots and diverges. The ideal rate changes throughout training.

Noisy gradients: Each batch of data gives a slightly different gradient. SGD follows every noisy signal, zig-zagging instead of heading straight to the optimum.

Same step size for all weights: Some weights need big updates, others need tiny nudges. SGD treats them all the same.

Learning Rate Schedules

One simple improvement: change the learning rate over time. The standard approach is warmup + cosine decay:

Learning Rate Schedule
1
Warmup (first ~1% of training): Start with a tiny learning rate and linearly increase it. This prevents the model from making huge, destructive updates with its initial random weights.
2
Cosine decay (rest of training): Smoothly decrease the learning rate following a cosine curve down to near zero. Early training needs big steps to find the right region; late training needs small steps for fine-tuning.

Momentum: The Rolling Ball

⚽

Imagine a ball rolling down a bumpy hill. Without momentum, the ball stops at every little bump and dip. With momentum, it builds up speed and rolls right over small bumps toward the real valley. SGD with momentum keeps a running average of past gradients, so the updates have "inertia" — consistent downhill directions get amplified, while noisy oscillations cancel out.

Momentum alone isn't enough for modern AI. We need optimizers that adapt the learning rate per parameter — which brings us to Adam and Muon in the next two chapters.

Key Insight

Training at scale requires more than basic gradient descent. Learning rate schedules (warmup → cosine decay) control how aggressive the updates are over time. Momentum smooths out noisy gradients by maintaining inertia. But the real workhorses are adaptive optimizers that adjust the step size for every single parameter individually.
