Chapter 10

Deep Networks: Hedgehogs and Quills

Why a single layer can only draw a straight line — and how adding layers and nonlinearity lets a model learn any pattern.

Before we crack open the transformer, we need a few more concepts. In Chapter 3 our turkey model had one layer — multiply and add. That can only produce a straight line. Real-world relationships are messy, curved, and complex. Let's see this with a new example.

The Alien Hedgehog Problem

Imagine you land on an alien planet with lots of hedgehogs. You measure the length of 1,000 of them and count their quills. Plot the data:

🦔
Hedgehog Data: Length vs Quills

The relationship is clearly not a straight line — it goes up, down, and up again. A simple linear model (like our turkey model) would draw a straight line through this mess and miss the pattern entirely.

Training vs Validation Data

Before we build models, an important concept: don't test on training data. If you use all 1,000 hedgehogs for training, the model might memorize the specific data points rather than learning the underlying pattern. This is called overfitting.

The fix: split the data into training (900 hedgehogs) and validation (100 hedgehogs). Train only on the 900, then check loss on the 100 the model has never seen. If training loss is low but validation loss is high — the model memorized instead of learned.

📈

Overfitting

Training loss: great. Validation loss: terrible. Model memorized the data, didn't learn the pattern.

📉

Underfitting

Both losses are high. Model is too simple to capture the pattern.

✅

Just Right

Both losses are low. Model learned the real pattern and generalizes to new data.

Model #1: Just Predict the Average

The simplest possible model: ignore the input entirely and always predict the average number of quills (3,030). No weights at all. This gives a Mean Squared Error (MSE) loss of about 1.05 (meaning we're off by about 1,025 quills on average).

Model #2: A Single Linear Layer

Like the turkey model but with one input (length) instead of two: prediction = w × length + b. After training, this draws the best possible straight line through the data. Loss drops to about 0.88 — better than the average, but still can only draw a straight line.

Model #3: Two Linear Layers — Still a Line!

Maybe adding a second layer will help? Input → Linear (1→10) → Linear (10→1). But here's the surprise: two linear layers in a row are mathematically equivalent to one linear layer. No matter how many linear layers you stack, you can only draw straight lines.

🔗

Why? A linear function of a linear function is still linear. If layer 1 does y = 3x + 2 and layer 2 does z = 5y + 1, then z = 5(3x + 2) + 1 = 15x + 11. That's still just one line. No matter how many layers: line × line × line = line.

The Breakthrough: Nonlinearity (ReLU)

The fix is breathtakingly simple. Between the linear layers, add a nonlinear activation function. The most popular one is ReLU:

ReLU — Rectified Linear Unit

ReLU(x) = max(0, x)

If x is positive → keep it. If x is negative → replace with zero. That's it.

Model #4: Linear → ReLU → Linear

Now we have: Input → Linear (1→10) → ReLU → Linear (10→1). This simple change is transformative. With ReLU between the layers, the model can learn any curved relationship. After training, the loss drops dramatically — the model can follow the up-down-up pattern of the hedgehog data.

The first linear layer takes the single input (length) and multiplies it by 10 different weights plus 10 different biases — producing 10 numbers. ReLU zeros out the negative ones. The second linear layer mixes these 10 numbers back into a single prediction. Total parameters: 10 weights + 10 biases + 10 weights + 1 bias = 31 parameters.

Going Deeper

More layers = more power to learn complex patterns. A model with 3 linear layers and ReLU between each can fit even more intricate shapes. But more layers also mean more parameters and more risk of overfitting — the model might memorize the training data's noise instead of learning the true pattern.

The trick is finding the right balance. For our hedgehog data, even 2-3 layers with ReLU are enough. For understanding all of human language? That takes 20, 32, or even 100+ layers — which is exactly what transformer models have.

Key Terminology

Parameters

Weights and biases together are called parameters. Our turkey model had 2 parameters (both weights). Model #4 has 31 parameters. A GPT model has hundreds of billions.

Linear Transformation (Green Box)

Multiplying by weights and adding biases: output = W × input + b. Going forward, we'll draw this as a green box and label it with dimensions like "1→10" meaning input size 1, output size 10. This is the building block of neural networks.

MLP (Multilayer Perceptron)

A stack of linear layers with activation functions (ReLU) between them. Model #4 is an MLP. You'll see MLPs inside every transformer block — they're where a lot of the "thinking within each position" happens.

Mean Squared Error (MSE)

Loss = average of squared errors. Better than sum (Chapter 3) because it lets you compare loss across different batch sizes. MSE of 4 means predictions are off by about √4 = 2 units on average.

Key Insight

Depth + Nonlinearity = Power. Linear layers alone can only compute straight lines, no matter how many you stack. Adding a nonlinear activation function (ReLU) between layers lets the model learn any pattern — curves, waves, and incredibly complex relationships. This is the foundation of "deep learning." The transformer is just a very clever arrangement of linear layers, activation functions, and one more crucial ingredient: attention (coming in Chapter 14).
