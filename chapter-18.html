Chapter 18

How Will We Know If the Model Works?

Validation loss, perplexity, and benchmarks — the three pillars of measuring AI model quality.

Validation Loss

During training we compute cross-entropy loss on the training data (to update weights) and on a separate validation set the model never trains on. This tells us if the model is truly learning or just memorising.

Reading the Tea Leaves

Both losses dropping: Genuine learning — keep training.

Training loss drops, validation loss rises: Overfitting! The model memorised training data quirks.

Both plateau: Model has learned all it can at this size. Need more parameters or better data.

Perplexity

Perplexity = 2loss. If loss is 3.0, perplexity is 2³ = 8 — the model is "as confused as if choosing between 8 equally likely options" at each token. An untrained model has perplexity near the vocab size (~65,536). A strong model gets down to 5–15.

LossPerplexityStage
11.02,048Random weights (untrained)
5.032Early training — basic patterns
3.08Decent model
1.52.8Strong model

Benchmarks

Loss measures general prediction. Benchmarks test specific capabilities: reading comprehension, math word problems (GSM8K), code generation (HumanEval), factual knowledge across 57 subjects (MMLU), logical reasoning (ARC), and common sense (HellaSwag). After each training phase we run all benchmarks and track improvements.

Key Insight

Quality is measured with validation loss (general prediction), perplexity (intuitive confusion), and benchmarks (specific skills). All three matter — a model needs both breadth and depth.
