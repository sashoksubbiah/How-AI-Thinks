Chapter 14

Attention

The heart of the transformer — how each token decides which earlier tokens to pay attention to, and how information flows between positions.

How Humans Pay Attention

Consider reading this paragraph word by word:

"In 1873, Tesla returned to his birthtown, Smiljan. Shortly after he arrived, Tesla contracted cholera; he was bedridden for nine months and was near death multiple times. Tesla's father promised to send him to the best engineering school if he recovered. What was the year when Tesla went back to Smiljan?"

With every word, your understanding refines. When you read "Tesla" after "1873," you know it's a person, not a car company (wrong era). When you read "he," you know it means Tesla. When the question asks for a year, you mentally reach back to "1873" at the start.

The model needs to do the same thing. When processing that final question mark, it needs to attend to "1873" from much earlier. The attention mechanism is how this happens.

The Query / Key / Value Framework

📚

Library analogy: You walk into a library with a question (your query). Each book has a label on its spine (its key) advertising what information it contains. You compare your query against each key. The books with matching keys — you pull them off the shelf and read their contents (their values).

In the transformer, every position emits three things (via learned linear transformations):

❓

Query (Q)

"What information am I looking for from earlier positions?"

🔑

Key (K)

"What kind of information do I have to offer?"

📦

Value (V)

"Here is my actual information."

Step-by-Step Attention Calculation

Let's walk through attention for "The capital of France is" with D=2 (simplified to 2 dimensions so we can visualize). After the Q, K, V linear transformations:

PositionTokenQueryKeyValue
1The—[0.91, 0.42][0.30, 0.10]
2capital—[-0.44, 1.34][0.80, -0.20]
3of—[-0.77, 1.18][0.10, 0.50]
4France—[-0.55, 1.27][0.60, 0.30]
5is[-0.51, 1.32][-0.63, 1.22][0.20, 0.40]

We focus on position 5 ("is"). Its query [-0.51, 1.32] will be compared against all keys.

Step 1: Compute Scores (Query · Key)

Multiply the query for position 5 element-wise with each key and sum (dot product). Higher score = query and key point in the same direction = more relevant.

Score = Query₅ · Key (dot product)
1
"The": (-0.51)(0.91) + (1.32)(0.42) = -0.46 + 0.55 = 0.09
2
"capital": (-0.51)(-0.44) + (1.32)(1.34) = 0.22 + 1.77 = 1.99 ← highest!
3
"of": (-0.51)(-0.77) + (1.32)(1.18) = 0.39 + 1.56 = 1.95
4
"France": (-0.51)(-0.55) + (1.32)(1.27) = 0.28 + 1.68 = 1.96
5
"is": (-0.51)(-0.63) + (1.32)(1.22) = 0.32 + 1.61 = 1.93

Step 2: Softmax → Attention Weights

Convert scores to probabilities using softmax (same function from Chapter 8). These are the attention weights — how much position 5 will "pay attention" to each earlier position:

Attention Weights for Position 5 ("is")

"is" pays 60% attention to "capital" — exactly what we'd want when trying to predict "Paris"!

Step 3: Weighted Sum of Values → Output

The output for position 5 is 60% of "capital"'s value + 26% of "France"'s value + 8% of "of"'s value + ... This blends information from the most relevant positions into position 5's representation.

Output = Σ (attention_weight × value)
→
0.60 × [0.80, -0.20] + 0.26 × [0.60, 0.30] + 0.08 × [0.10, 0.50] + ...
=
[0.48+0.16+0.01+..., -0.12+0.08+0.04+...] ≈ [0.66, -0.01]

Causal Masking: No Peeking!

During training, the full sequence is fed in at once (Chapter 8). But position 3 ("of") must not see positions 4 or 5 — it needs to predict the next token without cheating. The fix: set the attention scores for future positions to negative infinity before softmax. After softmax, these become exactly zero.

🎭
The Causal Mask
            The   cap   of   Fra   is
The   [✓    ✗    ✗    ✗    ✗]
cap   [✓    ✓    ✗    ✗    ✗]
of    [✓    ✓    ✓    ✗    ✗]
Fra   [✓    ✓    ✓    ✓    ✗]
is    [✓    ✓    ✓    ✓    ✓]

✓ = can attend, ✗ = blocked (−∞ before softmax → 0 after). Lower-left triangle only — each position can only look at itself and earlier positions. This is "causal" (can't see the future).

Multi-Head Attention

The model doesn't do attention just once — it does it multiple times in parallel, each with different learned Q, K, V transformations. These are called heads.

Why Multiple Heads?

Different heads can learn to focus on different things:

• Head 1 might track grammar (subject-verb agreement)
• Head 2 might track semantic meaning (what the sentence is about)
• Head 3 might track nearby words (local context)
• Head 4 might track distant references ("he" → "Tesla")

With D = 1280 and 10 heads, each head works in D/10 = 128 dimensions. The process: split Q, K, V into 10 chunks → do attention independently in each chunk → rejoin into one D-sized vector. No extra parameters for splitting/rejoining — it's just reshaping the tensors.

Note: splitting into heads, rotary embeddings (Chapter 15), and the actual dot-product/softmax attention computation all have zero learned parameters. The learning happens entirely in the four linear transformations (Q, K, V, output).

Key Insight

Attention is the mechanism that lets tokens look at each other. Each position generates a Query ("what do I need?"), every position offers Keys ("what do I have?") and Values ("here's my info"). High Query-Key matches mean high attention weights, which means more of that position's Value gets mixed in. Causal masking ensures no position can see the future. Multi-head attention lets the model attend to multiple different types of relationships simultaneously. This is the single most important innovation in the transformer.
