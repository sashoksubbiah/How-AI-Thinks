Chapter 24

Base Training

Phase 1: Feed the model trillions of tokens from the internet and teach it to predict language.

It's time. Everything we've learned — tokens, embeddings, attention, backpropagation, optimizers, GPUs — comes together now. We're going to train a 32-layer transformer model from scratch.

The Training Data

Base training uses a massive corpus of text scraped from the internet. Common Crawl, a non-profit, maintains free web crawl data. The training mix typically includes:

🌐

Web Text

Filtered web pages — the bulk of training data (~60-70%)

📚

Books

Published books across genres (~10%)

💻

Code

Open-source code from GitHub (~10-15%)

📰

Quality Text

Wikipedia, academic papers, curated sources (~10-15%)

The data is tokenized, shuffled, and packed into sequences of fixed length (e.g., 2048 tokens). Multiple documents may be concatenated into a single sequence, separated by special tokens.

What Happens During Training

The training loop runs continuously for days:

Load batch of 524,288 tokens
↓
Forward pass → predictions for all tokens
↓
Cross-entropy loss against actual next tokens
↓
Backward pass → gradients for all ~1.5B parameters
↓
Adam/Muon update weights → sync across 8 GPUs
↓ ~0.5 seconds per step, ~50,000 steps total ↓ repeat

As training progresses, the loss drops from ~11 (random guessing) down to ~3-4 (the model understands language). Periodically, we checkpoint the model (save all weights to disk) and run benchmark evaluations.

What the Base Model Can Do

After base training, the model can complete text fluently. Give it "The history of Chennai" and it'll write a decent paragraph about the city. But it's not yet a helpful assistant — it might continue your prompt with Wikipedia-style text, or repeat itself, or ignore your question entirely. Turning it into ChatGPT-like assistant requires the next phases: mid-training, supervised fine-tuning, and reinforcement learning.

Key Insight

Base training is where the model learns language and world knowledge — by predicting next tokens on trillions of words from the internet, books, and code. It runs for days on 8 GPUs, processing half a million tokens every half-second. The result is a model that understands language but isn't yet a helpful assistant.
